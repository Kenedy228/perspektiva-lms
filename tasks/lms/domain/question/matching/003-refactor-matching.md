# Доработка тестов и документации пакета matching

## Контекст

Необходимо доработать основной пакет `domain/question/matching`.

Важно: задача относится только к пакету `matching`.
Не нужно вносить изменения в подпакеты пакета `matching`.

---

# Цель 1. Переписать и расширить тесты пакета matching

Необходимо переписать или доработать тесты компонентов основного пакета `matching`.

Тесты должны покрывать:

* экспортируемые функции;
* экспортируемые методы;
* неэкспортируемые функции;
* неэкспортируемые методы;
* успешные сценарии;
* ошибочные сценарии;
* edge-case сценарии;
* доменные инварианты.

---

## Требования к тестам

* Использовать table-driven tests.
* Тесты должны быть читаемыми и поддерживаемыми.
* Не дублировать лишнюю setup-логику.
* Проверять не только факт ошибки, но и тип/причину ошибки через `errors.Is`, если в пакете используются sentinel errors.
* Проверять граничные значения.
* Проверять zero-value состояния, если они возможны.
* Проверять ограничения количества элементов, длины строк и идентификаторов, если такие инварианты есть в пакете.
* Не тестировать внутреннюю реализацию подпакетов.

---

## Ограничение по области изменений

Codex не должен изменять подпакеты:

```text
matching/answer
matching/pair
```

Если в проекте существуют другие подпакеты внутри `matching`, их тоже нельзя изменять.

Разрешено изменять только файлы основного пакета:

```text
internal/domain/question/matching/*.go
internal/domain/question/matching/*_test.go
```

Запрещено изменять:

```text
internal/domain/question/matching/*/*.go
internal/domain/question/matching/**/*_test.go
```

# Цель 2. Русифицировать ошибки основного пакета matching

Необходимо проверить ошибки в основном пакете `matching` и русифицировать их.

Требования:

* ошибки должны быть на русском языке;
* ошибки должны содержать полезный контекст;
* использовать wrapping через `%w`;
* не терять sentinel errors;
* не менять смысл ошибок.

Пример ожидаемого стиля:

```go
fmt.Errorf(
	"%w: количество пар не должно превышать %d штук (текущее количество - %d)",
	ErrInvalid,
	MaxPairs,
	len(pairs),
)
```

# Цель 3. Добавить godoc на русском языке

Необходимо добавить или обновить godoc-комментарии на русском языке для публичного API основного пакета `matching`.

Документация нужна для:

* экспортируемых структур;
* экспортируемых функций;
* экспортируемых методов;
* экспортируемых констант;
* экспортируемых ошибок.

Требования:

* комментарий должен начинаться с имени документируемой сущности;
* комментарий должен объяснять назначение и поведение;
* не писать формальные комментарии без смысла;
* не документировать неэкспортируемые детали без необходимости.

---

# Ограничения

* Не изменять подпакеты `matching`.
* Не изменять бизнес-логику, кроме русификации и детализации ошибок.
* Не добавлять новые абстракции.
* Не добавлять лишние интерфейсы.
* Не менять публичное API без необходимости.
* Не выполнять unrelated-рефакторинг.
* Сохранить существующий стиль проекта.
* Не использовать panic.

# Проверка результата

После выполнения задачи необходимо выполнить:

```bash
go test ./...
go vet ./...
```

---

# Ожидаемый результат

В результате выполнения задачи:

* тесты основного пакета `matching` переписаны или расширены;
* покрыты экспортируемые и неэкспортируемые функции/методы;
* ошибки основного пакета `matching` русифицированы и стали информативнее;
* добавлен или обновлён godoc на русском языке;
* подпакеты `matching` не изменены;
* проект успешно проходит `go test ./...` и `go vet ./...`.
